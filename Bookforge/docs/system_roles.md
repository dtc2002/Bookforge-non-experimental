# System Roles

## PopeBot / CrucibleBot
The implementation agent that reads plans, edits files in `Bookforge/`, runs validations, and opens pull requests.

PopeBot is not Bookforge.
PopeBot builds Bookforge.

## Bookforge
The software product being built.
Bookforge is a staged literary generation engine.

## Runtime Models
Language models called by Bookforge during operation, primarily through Ollama.

## Operator
The human supervising development, approving pull requests, and issuing overrides or direction changes.
