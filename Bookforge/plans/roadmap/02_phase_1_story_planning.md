# Phase 1 – Story Planning

## Goal

Convert a structured project brief into a usable, validated story plan.

## Purpose

This phase exists to lock narrative intent before prose generation begins.

Bookforge must not draft fiction directly from a raw prompt. It must first generate a planning stack that narrows creative freedom into a controlled structure.

## Inputs

- Project Brief
- Optional Universe Bible reference
- Optional operator notes

## Outputs

- Story Skeleton
- Beat Outline
- Chapter Plan
- Planning Assessment Report

## Sequence

1. Brief to Story Skeleton
2. Story Skeleton to Beat Outline
3. Beat Outline to Chapter Plan
4. Planning Review and Validation

## Story Skeleton contents

The Story Skeleton defines:

- working title
- genre
- subgenre
- target audience
- tone
- premise
- protagonist
- protagonist goal
- opposition force
- stakes
- central conflict
- story engine
- ending shape
- act structure
- forbidden elements
- required elements

## Beat Outline contents

The Beat Outline defines:

- opening condition
- inciting incident
- first major turn
- midpoint shift
- crisis
- climax
- resolution
- setup/payoff chains
- character arc movement
- emotional arc movement

## Chapter Plan contents

The Chapter Plan defines per chapter:

- chapter number
- working title
- chapter purpose
- pov
- location
- time context
- entry state
- exit state
- beats covered
- required callbacks
- constraints
- estimated length

## Validation goals

The planning stack must be checked for:

- schema validity
- internal consistency
- audience fit
- logical progression
- sufficient escalation
- setup/payoff coverage
- ending compatibility
- excessive complexity
- unresolved contradictions

## Non-goals

Phase 1 does not:

- write prose
- finalize chapter text
- invent unbounded side plots
- mutate canon without explicit instruction

## Acceptance criteria

Phase 1 is complete when:

- all required planning artifacts exist
- all planning artifacts pass schema validation
- the planning assessment reports no blocking errors
- a short demo brief can be processed end-to-end
